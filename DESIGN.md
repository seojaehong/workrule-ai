# AI Legal Desk DESIGN.md

## 1. Visual Theme & Atmosphere

- 제품 성격: 마케팅 랜딩이 아니라 `근거를 확인하며 일하는 법률 리서치 워크스페이스`
- 무드: 조용하고 정확하며 낮은 채도의 신뢰감
- 참고 방향: Cursor의 작업 중심 구조, Notion의 낮은 장식성, Linear의 정보 위계
- 핵심 원칙: 답변보다 `사건 개요`, `현재 질문`, `근거 인스펙터`가 먼저 보여야 한다

## 2. Color Palette & Roles

- `--background`: 따뜻한 오프화이트 계열의 작업 배경
- `--foreground`: 차콜 계열 본문 텍스트
- `--muted`: 설명/메타 정보
- `--accent`: 링크, 포커스, 상태 강조용 블루
- `--line`: 얇은 경계선, 분할선
- `--panel`: 반투명 작업 패널

색은 많지 않아야 한다.

- 기본은 무채색 + 블루 한 가지
- 성공/위험 상태는 상태 전달용으로만 사용
- 넓은 그라데이션, 화려한 광택, 과한 글로우 금지

## 3. Typography Rules

- 기본 폰트: Geist Sans
- 보조 폰트: Geist Mono
- 헤드라인은 짧고 밀도 있게
- 제품 UI에서는 문장보다 라벨, 섹션명, 메타 정보가 중요하다
- 과한 대문자 남용 금지. 단, 작은 kicker와 상태 라벨은 허용

## 4. Component Stylings

- 카드 남용 금지
- 가능하면 `border + spacing + alignment`로 구조를 만든다
- 버튼은 기능적이어야 하며 과한 배경 효과를 넣지 않는다
- 입력창은 넓고 조용해야 하며, 법률 서비스답게 안정감 있는 밀도를 유지한다
- 우측 인스펙터는 답변의 부속이 아니라 근거 확인을 위한 핵심 패널이다

## 5. Layout Principles

- 기본 레이아웃은 3열
  - 좌: 사건 브리프
  - 중: 대화 작업면
  - 우: 근거 인스펙터
- 첫 화면부터 실제 작업이 가능해야 한다
- 큰 랜딩 히어로, 장식용 통계 바, 과도한 카드 그리드는 피한다
- 모바일에서는 순서를 `브리프 -> 대화 -> 근거`로 자연스럽게 접는다

## 6. Depth & Elevation

- depth는 최소화
- 표면 구분은 얇은 테두리, 반투명 패널, 넉넉한 여백으로 해결
- 떠다니는 카드나 두꺼운 그림자 금지

## 7. Do's and Don'ts

### Do

- 작업 시작점이 바로 보이게 만든다
- “무엇을 할 수 있는지”보다 “어디서 시작하는지”를 먼저 보여 준다
- 근거, 출처, 도구 흐름을 분리해 스캔 가능하게 한다
- 법률 자문 오인 방지 문구는 항상 보이게 둔다

### Don't

- 랜딩 페이지처럼 보이게 만들지 않는다
- 보라색/네온 계열 AI UI 패턴을 쓰지 않는다
- 섹션마다 카드 상자를 두르지 않는다
- 의미 없는 아이콘, 배지, 수식어를 추가하지 않는다

## 8. Responsive Behavior

- 데스크톱: 3열 유지
- 태블릿: 2열 또는 세로 적층
- 모바일: 사건 개요 입력과 질문 입력이 항상 먼저 보여야 한다
- 모바일에서 우측 인스펙터는 답변 아래로 내려오되 제목과 메타는 유지한다

## 9. Agent Prompt Guide

- “Cursor / Notion / Linear 계열의 조용한 업무용 UI”
- “마케팅 랜딩처럼 만들지 말고 실무 도구처럼 구성”
- “좌측 브리프, 중앙 대화, 우측 근거 인스펙터”
- “무채색 기반, 블루 한 가지 강조색, 얇은 구분선, 낮은 그림자”
